    Road Map

    First why choose Lua? Lua is:

    • Very small
    • Extremely powerful
    • Very easy to learn for people from a non programming background or profession because:
      • It has a simple syntax
      • It has a single data structure
    • Well proven language of choise in well known projects like:
      • Mediawiki (Wikipedia)
      • Wireshark
      • World of Warcraft
      • LuaTex
      • NMAP
      • Celestia
      • and so on...
    These reasons by themself make it a very powerful language for newcomers to adopt to get things done.

    • Lua-GL finally modularizes the design of complex software in a language that can be adapted by non expert programmers.
      • Most graphical software are mamoth code bases in languages such as C/C++/.NET/Java which are comparitively harder to adopt than languages like Python, Ruby and Lua. On top of that they don't incorporate modularity and flexibility by abstracting away the design into multiple layers.
    • Lua-GL aims to provide the 1st level library in a well chosen simple scripting language that can provide all the mechanisms to create complex graphical software
      • Software like a electronic schematic editor or a block diagram creator software similar to Microsoft Visio. It abstract away all the complex graphical operations and provides a simple extensible API that can be used to create a software with custom behavior, functionality and workflow.
    • Lua-GL even goes a step further by abstracting away the underlying GUI framework
      • to make it easy to swap it for say QT or even Javascript routines now that we can run Lua directly on the browser.
    With these design choices for Lua-GL it simply makes things a lot easier, faster, interoperable and more scalable for projects using it.
